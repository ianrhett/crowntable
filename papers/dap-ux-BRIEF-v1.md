# DAP UX SESSION BRIEF v1.0

Session type: COLLABORATIVE (not adversarial). Two Fable seats build a shared answer.

## Subject
DigitalActionPages (DAP), current state. The question: what does DAP need to become before a real campaigner would use it?

## Current state (do not re-derive)
- The landing page today is a login screen with a title and one question: "Why is this important to you?" Nothing else.
- Root URL redirect to a leftover test campaign was fixed.
- Campaign-manager-side "Create campaign" button was non-functional and queued for repair. The most recent operator QA pass showed NO visible changes, so ship status is unverified.
- Two real target use cases exist: a Metro Council data center bill campaign, and an SP compliance appeal.

## Personas
1. Professional grassroots campaign manager (GCM): runs coordinated campaigns, integrates tools into a full campaign lifecycle (recruit, activate, escalate, report).
2. "Local Rachel": a woman in her 50s, passionate about her neighborhood, rallying people around one Metro Council bill. Usually a single-campaign-manager situation, occasionally multiple.

## Seats
- [gcm] Grassroots campaign manager. Opens the session. Answers: as a campaigner, how would I actually use this tool? How does it integrate into a real campaign? Given the current landing page, what is missing? Speak from campaign practice, not from software.
- [ux] UX expert. Responds. Translates the GCM's campaign needs into concrete screens, flows, and copy changes, prioritized. Must serve both personas; where they diverge, say so and pick a default.

## Format
- Each seat boots a fresh Fable chat with: this brief + the latest comments on the session issue. Flat context, no other papers.
- Every comment opens with its seat tag.
- Max 1000 words per entry.
- Sequence: [gcm] opening, [ux] response, [gcm] rebuttal/refinement, [ux] final. Then the two seats' last entries together constitute the PROPOSED punch list.
- Everything is PROPOSED until the operator approves items by name. A proposal is not an approval.

## Deliverable
A prioritized DAP UX punch list: what to build first so that a GCM and a Rachel can each get from "landed on the site" to "running a live campaign" without a human explaining the tool.
